Three questions, deliberately. The answers steer how the pipeline writes and pitches the lecture — not just what topic to fetch.

1
TopicThe subject to teach. Free text — drives triage and source retrieval.
2
Starting level"Name what you know and what you don't." Sets the level anchor so jargon is introduced at the right depth.
3
GoalQuick-pick pills (Intuition / Implement / Use / Interview) or a custom goal. Shapes task emphasis.
4
Pipeline previewSets expectations before committing: triage → retrieve → teach → decompose → verify → skeptic.
5
Validation gate"Start verifying" stays disabled until all three fields are filled; the hint text explains what's missing.

Reference

States & rules across the flow

New topic · empty
All fields blank. Submit disabled, hint reads "Fill in all three fields to continue".
New topic · ready
All three valid → button enabled, hint flips to "Ready — click to start verifying".
Library card · running
New topic inserted at top with a pulsing dot and its current pipeline stage as the subtitle.
Pipeline · in progress
Stages advance over time; timer counts up; header shows "Verifying…".
Pipeline · done
Header → "Ready", timer → "done · Ns". Card in Library flips to ready with a trust bar.
