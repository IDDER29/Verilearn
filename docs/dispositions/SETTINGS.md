### Settings, Profile & Privacy (SETTINGS)

Honest, evidence-based disposition of the SETTINGS domain against the current codebase. Key finding: the settings shell and all seven sub-panels are built and route correctly, and two panels (Profile, Plan) read real user/topic data — but there is **no per-user preferences store**, **no settings `use server` mutation** (only `conflict/task/review/auth/topic-actions.ts` exist), and no `frozen`/preferences/timezone fields on `User` (`web/lib/store/entities.ts:14`). So every toggle, slider, Save/Reset, and danger-zone button is presentational; nothing persists.
