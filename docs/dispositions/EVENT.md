### Events: Workshops, Groups & Challenges (EVENT)

Evidence baseline: the Events domain is present only as three static, hardcoded UI screens — `/home/claude/repo/web/app/events/page.tsx` (hub), `/home/claude/repo/web/app/events/[slug]/page.tsx` (detail; single hardcoded route), and `/home/claude/repo/web/app/events/registered/page.tsx` (confirmation). There is **no event/registration/challenge data model** (`web/lib/store/entities.ts` has no event entity), **no events service** (`web/lib/services/` has none; the only `event` references are `VerificationEvent` trust-ledger records, unrelated), and **no tests**. Registration is a plain `<Link>` navigation; the challenge ring (6/7), going-counts, "Reminder set" chip, and "Add to calendar" button are decorative. Per the roadmap this domain is R2/R3, "no data model seeded." Some destination engines the domain routes *into* do exist and are wired (Conflicts, Gap Map, Sources/SME, FSRS/Review, Progress signals, RBAC, tenant scoping, COPPA age-gate).
