### Accessibility, Mobile & Offline (A11Y)

Honest engineering disposition of the A11Y domain against the current codebase. This is a cross-cutting, non-functional domain (accessibility, responsive-mobile, PWA/offline). The one real building block present is the canonical non-color trust encoding contract (`lib/domain/types.ts` `TRUST_LABEL` → `{ label, glyph }` for all six states, consumed across several server components) plus the engine-level seams (commit-before-reveal review, FSRS reschedule persistence, RBAC epistemic firewall). Almost none of the accessibility semantics, responsive-mobile shell, or offline/PWA engine is actually built: no ARIA/landmarks/skip-link (only `role="button"` on two spans in `components/workspace/LectureTab.tsx`), no `@media`/responsive layout (fixed inline-style desktop UI), no `prefers-reduced-motion`, no accommodation/extended-time model, no service worker / offline queue / sync backend, no accessibility statement / VPAT, no captioning.
